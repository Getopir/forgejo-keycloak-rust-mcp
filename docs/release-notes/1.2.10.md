# Release 1.2.10

## Changes

- Validates every token-signing JWK before daemon startup completes.
- Allows only approved asymmetric JWT algorithm and key-type combinations.
- Rejects RSA signing keys below 2048 bits, symmetric JWT keys, unsupported or
  mismatched curves, missing algorithms or key IDs, and duplicate signing IDs.
- Requires ES256 with P-256, ES384 with P-384, and EdDSA with Ed25519.
- Binds token validation to the selected JWK algorithm instead of trusting the
  JWT header algorithm.
- Adds focused regression tests and updates the JWKS rotation documentation,
  OpenSSF evidence, and maintainer backlog.

No publicly known vulnerability was fixed in this release.

## Install

```sh
cargo install forgejo-keycloak-rust-mcp --version 1.2.10 --locked
```
