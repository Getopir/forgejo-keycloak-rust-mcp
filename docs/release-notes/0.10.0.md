# Release 0.10.0

`0.10.0` adds approval-backed release creation to the Phase 2 Forgejo MCP gateway.

## Added

- `create_release` now has an executable path after approval validation.
- Release creation supports dry-run preview without mutating Forgejo.
- Release options are bound to the approval body hash, so the executed payload must exactly match the approved payload.
- Approval records are consumed before the Forgejo release call, so replay is denied.
- The executor must be a different mapped principal from the approver.
- `forgejo-mcpctl create-release` wraps the release operation for shell-based agent and operator workflows.

## Release Options

`create_release` accepts JSON in the `body` request field.

Required:

- `tag_name`

Optional:

- `target_commitish`
- `name`
- `body`
- `draft`
- `prerelease`
- `hide_archive_links`

## Still Disabled

- Repository deletion.
- Admin and organization mutation tools.
- Generated Forgejo API coverage.
- Release asset upload, release deletion, and release replacement.
