# Release 0.6.0

`0.6.0` is the Phase 1 hardening and Phase 2 baseline release.

## Added

- Duplicate Keycloak `(issuer, subject)` mapping rejection.
- Required-field validation for principal map entries.
- `api_token_env` validation so mappings only name environment variables using ASCII letters, digits, and underscore.
- Spoofed trusted identity header rejection on `/mcp`.
- Server-capped `limit` and page-token `cursor` support for list operations.
- `list_repository_issues`.
- `list_pull_requests`.
- `list_pull_request_reviews`.
- `list_releases`.
- `list_notifications`.
- Executable additive `create_issue_comment` for issue and pull-request conversation comments.
- Approval-required response behavior for high-risk operations without an approval ID.

## Security

- Caller-supplied trusted identity headers are denied before Forgejo delegation.
- Principal-map ambiguity is denied at startup.
- Token values remain outside source-controlled mapping files and are read only from runtime environment variables.
- List responses return bounded summaries instead of full Forgejo objects.

## Not Included

- Persistent approval records.
- Pull-request merge execution.
- Release creation or publication execution.
- Repository deletion or admin execution.
- Generated Forgejo API coverage.
