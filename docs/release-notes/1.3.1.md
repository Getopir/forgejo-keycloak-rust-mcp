# Release 1.3.1

## Changes

- Makes approval consumption atomic across concurrent gateway instances by
  combining a process mutex with an OS-backed lock around validation and the
  consumed-record append.
- Adds a concurrency regression test proving that exactly one simultaneous
  consumer can use an approval.
- Retains JWK-pinned JWT algorithm validation and adds direct regression
  coverage for token-header and signing-key algorithm mismatches.
- Adds configurable outbound Forgejo connect and total request timeouts,
  defaulting to 5 and 30 seconds, with bounded stalled-response coverage.
- Updates maintained configuration and wiki documentation for these controls.

No publicly known vulnerability with an assigned CVE was fixed in this release.

## Install

```sh
cargo install forgejo-keycloak-rust-mcp --version 1.3.1 --locked
```
