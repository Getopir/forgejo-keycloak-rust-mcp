# Release 2.1.0

## Bounded Branch Status

Version `2.1.0` adds the first semantic operation after the Forgejo 16
compatibility baseline: `get_branch_status`.

- Accepts `owner/repository@branch` and
  `forgejo://branch/owner/repository/branch` targets.
- Uses the mapped Forgejo principal and `forgejo:repo:read`; Forgejo remains the
  final repository ACL authority.
- Requires no approval because the operation is read-only.
- Returns bounded branch, commit, protection, merge/push permission,
  required-context, and combined commit-status summaries.
- Caps required contexts and statuses at 50 items each, individual strings at
  documented byte limits, the branch response at 64 KiB, and the combined
  status response at 256 KiB, with a final 320 KiB operation-result ceiling.
- Uses the existing configurable Forgejo connect and total request timeouts.
- Rejects malformed targets, unknown envelope fields, and operation-inapplicable
  fields before calling Forgejo.
- Records structured allow, deny, downstream-failure, and success audit events
  without response bodies or credentials.

The reviewed Forgejo 16 catalog now contains 20 endpoint overlays representing
18 named semantic operations. The remaining 486 endpoints stay disabled.

## Upgrade Impact

No configuration or state migration is required from `2.0.0`. The release
continues to require Forgejo semantic core version `16.0.0`; build metadata such
as `16.0.0+gitea-1.22.0` remains accepted.

No publicly known vulnerability with an assigned CVE was fixed in this
release.

## Install

```sh
cargo install forgejo-keycloak-rust-mcp --version 2.1.0 --locked
```
