# Release 2.0.0

## Compatibility Change

Version `2.0.0` starts the Forgejo 16-only release line. When `--forgejo-url`
is configured, the daemon reads the public Forgejo `/api/v1/version` endpoint
before binding its listener and requires an exact `16.0.0` semantic core
version. Forgejo build metadata such as `16.0.0+gitea-1.22.0` is accepted.

Older, newer, prerelease, malformed, unreachable, or non-successful version
responses stop startup with a clear compatibility error. Deployments without a
configured Forgejo URL retain metadata and policy-only operation, but cannot
execute Forgejo-backed tools.

## Verification And Scope

- Regenerates the reviewed API coverage catalog from the pinned Forgejo
  `16.0.0` Swagger document: 506 operations, 18 endpoint overlays representing
  17 existing named semantic operations, and 488 disabled metadata-only
  operations.
- Adds contract coverage proving every existing semantic operation maps to the
  Forgejo 16 document.
- Adds startup-probe coverage for accepted build metadata and rejected older,
  newer, prerelease, and malformed versions.
- Exposes `required_forgejo_version` and `verified_forgejo_version` in
  `/health` for deployment readback.
- Adds no new executable MCP operation and does not enable any of the 15
  endpoints added in Forgejo 16.
- Requires no configuration or state-file migration from `1.3.1`; operators
  must upgrade Forgejo to `16.0.0` before upgrading the gateway.

No publicly known vulnerability with an assigned CVE was fixed in this
release.

## Install

```sh
cargo install forgejo-keycloak-rust-mcp --version 2.0.0 --locked
```
