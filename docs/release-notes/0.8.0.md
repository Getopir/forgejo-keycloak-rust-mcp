# Release 0.8.0

`0.8.0` hardens Phase 2 approval gates with persistent exact-payload approval records.

## Added

- `create_approval` MCP operation protected by `forgejo:approval:grant`.
- File-backed append-only JSONL approval store configured with `FORGEJO_MCPD_APPROVAL_STORE`.
- Approval TTL configured with `FORGEJO_MCPD_APPROVAL_TTL_SECONDS`, defaulting to 900 seconds.
- SHA-256 body binding for approval records.
- Principal binding across Keycloak issuer, subject, OAuth client, mapped Forgejo login, and optional Forgejo user ID.

## Security

- Approval IDs are no longer accepted as caller-supplied hints.
- Missing, malformed, unknown, expired, revoked, wrong-operation, wrong-target, wrong-state, wrong-body, and wrong-principal approvals are denied.
- High-risk operations still do not execute in this release. A valid approval returns an accepted non-executing gate response until the concrete mutation tool is implemented and tested.
- Approval files must be stored outside the repository and readable/writable only by the gateway service account.
