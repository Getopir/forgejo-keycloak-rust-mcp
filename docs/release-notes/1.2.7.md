# Release Notes - 1.2.7

`1.2.7` adds bounded per-agent admission control and publishes project-readiness evidence without expanding the executable Forgejo operation surface.

## Changes

- Adds in-memory token buckets for enabled mapped agents, keyed by normalized Keycloak `(issuer, subject)`.
- Adds configurable request capacity, refill window, and maximum tracked-agent bounds with positive-value startup validation.
- Returns HTTP `429 Too Many Requests`, `Retry-After`, and a structured retry value when an agent exhausts its bucket.
- Records rate-limit denials through the structured audit path and fails closed when the tracked-agent bound is full.
- Documents process-local reset and multi-instance limitations and retains reverse-proxy limits as a deployment requirement.
- Publishes `.bestpractices.json` with evidence-backed OpenSSF passing-series proposals while leaving external self-certification to an authenticated maintainer.
- Adds an administrator-only Forgejo-linked MCP settings surface to the maintained backlog, with explicit secret-isolation, validation, audit, reload, and rollback requirements.

## Install

```sh
cargo install forgejo-keycloak-rust-mcp --version 1.2.7 --locked
```

No MCP operation, scope, approval requirement, or executable Forgejo capability was added in this release.
