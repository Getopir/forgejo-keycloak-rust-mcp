# Release 0.7.0

`0.7.0` completes the remaining Phase 2 repository tasks for resource identifiers and CLI wrappers.

## Added

- Stable `forgejo://...` resource URIs in repository, issue, pull-request, review, release, notification, and comment summaries.
- Resource URI target parsing for repository and numbered issue or pull-request targets.
- `forgejo-mcpctl`, a CLI wrapper for curated MCP operations.
- CLI commands for gateway probe, repository metadata, issues, comments, pull requests, reviews, releases, and notifications.
- `deny.toml` for dependency advisory, license, source, and duplicate-version policy checks.
- `docs/security-checks.md` with local `cargo-audit`, `cargo-deny`, and SBOM instructions.
- Optional `.forgejo/workflows/ci.yml` for self-hosted Forgejo Actions runners.

## Security

- The CLI reads the bearer token from an environment variable named by `FORGEJO_MCPCTL_TOKEN_ENV`, defaulting to `ACCESS_JWT`.
- The CLI does not accept raw bearer tokens as command-line arguments.
- High-risk operations remain non-executing until persistent approval validation is implemented.
- `cargo-audit` completed successfully against the lockfile during this pass.
- `cargo-deny` and `cargo-cyclonedx` were not installed before the tool-install command timed out; their configs/docs are present for CI or a later local run.

## Lab Status

Live Phase 2 MCP testing is still pending. During this release pass, Forgejo itself was verified through HTTP version readback and SSH Git read/write using a temporary branch, but `192.168.87.91:7080` was not listening for `forgejo-mcpd`.
