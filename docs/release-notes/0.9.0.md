# Release 0.9.0

`0.9.0` makes pull-request merge the first approval-backed high-risk Forgejo operation.

## Added

- Single-use approval consumption before high-risk execution.
- Approver/executor separation: the mapped principal that creates an approval cannot execute it.
- `merge_pull_request` dry-run preview with no Forgejo mutation.
- Approval-backed `merge_pull_request` execution through the mapped executor's Forgejo token.
- CLI commands for `create-approval` and `merge-pull-request`.

## Security

- Consumed approvals cannot be replayed.
- Missing, malformed, expired, revoked, consumed, mismatched, and same-principal approvals are denied.
- Merge execution audits before approval consumption and after Forgejo returns.
- Release publication, admin actions, and destructive operations remain disabled.
