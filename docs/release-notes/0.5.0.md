# Release 0.5.0

`0.5.0` is the first Phase 1 beta release.

## Added

- Explicit principal mapping from Keycloak `(issuer, subject)` to Forgejo account metadata.
- Optional local principal-map JSON file loaded with `FORGEJO_MCPD_PRINCIPAL_MAP`.
- Read-only `list_repository_metadata` MCP operation backed by Forgejo API.
- Per-principal Forgejo API token lookup through an environment variable named by the mapping entry.
- Trusted reverse-proxy header derivation from the mapped Forgejo principal.
- Tests for mapping behavior, disabled mappings, trusted header construction, repository target parsing, and bounded metadata conversion.

## Security Notes

- The gateway still validates Keycloak issuer, audience, expiry, signature, and required claims before policy or Forgejo execution.
- Missing OAuth scope is denied before any Forgejo call.
- Unknown or disabled principal mappings are denied before any Forgejo call.
- Forgejo API tokens are not stored in the mapping file; the file stores only environment variable names.
- Mutating Forgejo operations remain disabled.

## Not Yet Included

- Issue, pull request, review, release, or notification execution.
- Approval gates for mutating actions.
- Generated Forgejo API coverage.
