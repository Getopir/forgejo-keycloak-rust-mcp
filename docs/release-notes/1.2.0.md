# Release Notes - 1.2.0

`1.2.0` hardens Forgejo MCP pull-request source-authority hygiene for governed agent workflows.

Highlights:

- Persists authoritative PR readback after PR create and merge paths, including PR number, head SHA, state, merged state, merge commit SHA, combined check state, branch-ref existence, and stale classification.
- Reads sparse PR create responses back by base/head first, then falls back to bounded open-PR matching by repo, head, base, and title.
- Detects open PRs with no commits and no changed files ahead of base, comments on them, and closes them as stale instead of reporting unfinished work.
- Checks PR head status before merge and reports exact non-success contexts with status and target/status URLs.
- Adds `status_check_wait_seconds` and `status_check_poll_seconds` merge options, including CLI support.
- Adds structured MCP gateway request/readback logs with operation, request ID, actor, target, PR number, response shape, and readback result without logging secrets.

Install:

```sh
cargo install forgejo-keycloak-rust-mcp --version 1.2.0 --locked
```
