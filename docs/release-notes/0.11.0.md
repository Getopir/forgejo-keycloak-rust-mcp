# Release 0.11.0

`0.11.0` starts Phase 3 by adding generated Forgejo API classification
coverage without enabling generic endpoint forwarding.

## Added

- Pinned the live Forgejo `15.0.3+gitea-1.22.0` Swagger document under
  `vendor/forgejo-api`.
- Added provenance and SHA-256 for the pinned API spec.
- Added generated endpoint classification in the policy crate.
- Classified all 491 pinned Forgejo API operations by target type, risk,
  approval requirement, and exposure.
- Added `forgejo_api_coverage` as a bounded metadata-only MCP operation.
- Added `forgejo-mcpctl api-coverage` for shell-based agent and operator
  readback.
- Generated `docs/generated/forgejo-api-coverage.md`.

## Safety

- Generated coverage does not enable arbitrary Forgejo API execution.
- Only endpoints mapped to reviewed semantic MCP operations are exposed.
- All other generated endpoints remain disabled metadata until they receive a
  reviewed operation, scope, risk class, output limit, and approval policy.
- Destructive and admin endpoints are classified as approval-required and stay
  disabled.
