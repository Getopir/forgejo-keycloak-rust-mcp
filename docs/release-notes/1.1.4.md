# Release Notes - 1.1.4

`1.1.4` fixes the governed Forgejo MCP pull-request creation response contract.

Highlights:

- Keeps the existing approval-backed Forgejo `create_pull_request` mutation.
- Normalizes successful PR creation responses so `result.pull_request` contains a PR number, URL, state, title, head metadata, base metadata, and mergeability when Forgejo exposes it.
- Falls back to authoritative Forgejo readback when the create response is sparse or empty.
- Matches readback candidates by repository, open state, exact head/base, and title as a tie-breaker.
- Fails loudly on no-match or ambiguous readback instead of returning a successful response without a PR number.
- Keeps secret values out of readback errors and logs.

Install:

```sh
cargo install forgejo-keycloak-rust-mcp --version 1.1.4 --locked
```
