# Contributing

Contributions from people and AI-assisted coding workflows are welcome.

1. Describe the problem before starting a large change.
2. Keep the change focused and add tests for behavior changes.
3. Update repository and wiki documentation when public behavior changes.
4. Run `cargo fmt --check`, `cargo test --workspace`, and `reuse lint`.
5. Open a pull request describing the change and verification.

AI-generated and AI-assisted changes are accepted. Contributors must review the
result, confirm its license and origin, run the tests, and remain responsible
for the submitted code.

Keep the project clean-room and never submit secrets, tokens, private keys,
internal addresses, or production credentials. Security issues must be reported
privately according to `SECURITY.md`.

The repository root contains the full `CONTRIBUTING.md` and the coding-agent
instructions in `AGENTS.md`.
