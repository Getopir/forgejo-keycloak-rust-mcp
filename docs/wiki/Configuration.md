# Configuration

Configure the daemon with these values:

- Keycloak issuer.
- OIDC discovery URL.
- MCP audience.
- Public MCP resource URL.
- Listen address.
- Forgejo base URL for Forgejo-backed tools.
- Principal map path.
- Maximum page limit for bounded list responses.
- Approval store path for short-lived high-risk operation approvals.
- Approval TTL in seconds.

Use environment variables or a deployment secret file outside the repository. Never commit tokens, client secrets, or private keys.

The principal map binds Keycloak `(issuer, subject)` to a Forgejo login and names the environment variable that contains that mapped user's Forgejo token. The map must not contain token values. Duplicate mappings and malformed token environment names are rejected at startup.

Configured trusted identity headers, such as `X-WEBAUTH-USER`, are generated by the gateway. Requests that supply those headers directly to `/mcp` are rejected as spoof attempts.

Set `FORGEJO_MCPD_APPROVAL_STORE` to enable persistent approval validation. The approval store is append-only JSONL and must live outside the repository, outside web roots, and under filesystem permissions limited to the gateway service account.
